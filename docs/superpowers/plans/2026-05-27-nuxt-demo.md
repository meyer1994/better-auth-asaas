# Nuxt Demo App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A minimal-functional Nuxt 4 demo of `better-auth-asaas` with `/login`, `/signup`, and `/payments` pages, plus the prerequisite fixes to make the example actually run.

**Architecture:** Nuxt 4 app under `examples/nuxt/`. Better-auth handler already mounted at `server/api/auth/[...all].ts`. The asaas plugin is configured server-side; the Vue client uses better-auth/vue plus the asaas client plugin. Auth guards live inline in each page's `<script setup>` — no route middleware files. `/payments` keeps a local-storage list of created payment IDs and re-fetches each one via the package's `getPayment` endpoint (the package has no list endpoint).

**Tech Stack:** Nuxt 4, Vue 3 `<script setup>`, Nuxt UI 4 (`UForm`, `UTable`, `useToast`), better-auth 1.6, `better-auth-asaas` (workspace), VueUse `useLocalStorage`, drizzle + better-sqlite3 (already wired).

**Spec reference:** `docs/superpowers/specs/2026-05-27-nuxt-demo-design.md`

**Spec deviation note:** Spec prerequisite #3 (the `server/api/auth/[...all].ts` catch-all) already exists. The other two prerequisite fixes still apply.

**No tests:** Per the spec ("No tests for the example") and the user's "Minimal functional" choice, this plan does not include automated tests. Verification is manual via `pnpm dev` and a browser.

## Task 1: Fix the server-side auth config

The example's `server/utils/auth.ts` imports `charge` and `webhooks` from `better-auth-asaas`, but the current package only exports `asaas`. It also passes `createCustomerOnSignUp` and `use: [...]` to `asaas()`, but the current `Options` type accepts only `apiKey` and `sandbox`.

**Files:**
- Modify: `examples/nuxt/server/utils/auth.ts`

- [ ] **Step 1: Rewrite the file**

Replace the entire contents of `examples/nuxt/server/utils/auth.ts` with:

```ts
import { betterAuth } from 'better-auth'
import { drizzleAdapter } from '@better-auth/drizzle-adapter'
import { asaas } from 'better-auth-asaas'
import { db } from '../db'
import * as schema from '../db/schema'

export const auth = betterAuth({
  secret: process.env.BETTER_AUTH_SECRET!,
  baseURL: process.env.BETTER_AUTH_URL ?? 'http://localhost:3000',

  database: drizzleAdapter(db, { provider: 'sqlite', schema, debugLogs: true }),

  emailAndPassword: { enabled: true },

  user: {
    additionalFields: {
      cpfCnpj: {
        type: 'string',
        required: true,
        input: true,
      },
    },
  },

  plugins: [
    asaas({
      apiKey: process.env.ASAAS_API_KEY!,
      sandbox: process.env.ASAAS_SANDBOX !== 'false',
    }),
  ],
})
```

Why the `user.additionalFields` block: better-auth's `signUp.email` only forwards fields it knows about. Without declaring `cpfCnpj` here, the value sent from the signup form is dropped before the asaas plugin's `databaseHooks.user.create.after` hook runs, and the customer is never created in Asaas. The DB column already exists in `server/db/schema.ts`.

## Task 7: Add `/payments` page

This is the largest task. It implements: inline auth guard, header with sign-out, create-charge form, and a table backed by `useLocalStorage('asaas:payment-ids', [])` that re-fetches each payment via `auth.asaas.payments.get`.

**Important — client method shape:**
The asaas client plugin only declares `$InferServerPlugin`, so better-auth derives method names from the server endpoint paths. The expected derived shape, based on paths like `/asaas/payments`, `/asaas/payments/get`, `/asaas/payments/delete`:

- Create: `auth.asaas.payments(...)` or `auth.asaas.payments.$post(...)` — better-auth typically exposes the POST-to-collection as the bare segment.
- Get: `auth.asaas.payments.get({ id })`
- Delete: `auth.asaas.payments.delete({ id })`

If at implementation time the actual derived names differ (log `auth.asaas` in the browser console to inspect), adjust the three call sites in this file accordingly. Do **not** wrap them in the package — that's out of scope.

**Files:**
- Create: `examples/nuxt/app/pages/payments.vue`

- [ ] **Step 1: Create the file**

```vue
<script setup lang="ts">
import { z } from 'zod'
import type { FormSubmitEvent, TableColumn } from '@nuxt/ui'

const auth = useAuth()
const toast = useToast()

const { data: session } = await auth.useSession(useFetch)
if (!session.value) {
  await navigateTo('/login', { replace: true })
}

const paymentIds = useLocalStorage<string[]>('asaas:payment-ids', [])

type PaymentRow = {
  id: string
  value: number
  status: string
  billingType: string
  dueDate: string
}

const rows = ref<PaymentRow[]>([])
const tableLoading = ref(false)

async function fetchOne(id: string): Promise<PaymentRow | null> {
  const { data, error } = await auth.asaas.payments.get({ id })
  if (error || !data) {
    toast.add({ title: `Failed to fetch ${id}`, description: error?.message, color: 'error' })
    return null
  }
  return {
    id: data.id,
    value: data.value,
    status: data.status,
    billingType: data.billingType,
    dueDate: data.dueDate,
  }
}

async function refreshAll() {
  tableLoading.value = true
  const results = await Promise.all(paymentIds.value.map(fetchOne))
  rows.value = results.filter((r): r is PaymentRow => r !== null)
  tableLoading.value = false
}

onMounted(refreshAll)

async function refreshRow(id: string) {
  const row = await fetchOne(id)
  if (!row) return
  const idx = rows.value.findIndex(r => r.id === id)
  if (idx >= 0) rows.value[idx] = row
}

async function deleteRow(id: string) {
  const { error } = await auth.asaas.payments.delete({ id })
  if (error) {
    toast.add({ title: 'Delete failed', description: error.message, color: 'error' })
    return
  }
  paymentIds.value = paymentIds.value.filter(p => p !== id)
  rows.value = rows.value.filter(r => r.id !== id)
  toast.add({ title: 'Payment deleted', color: 'success' })
}

const columns: TableColumn<PaymentRow>[] = [
  { accessorKey: 'id', header: 'ID' },
  { accessorKey: 'value', header: 'Value' },
  { accessorKey: 'status', header: 'Status' },
  { accessorKey: 'billingType', header: 'Billing' },
  { accessorKey: 'dueDate', header: 'Due' },
  { id: 'actions', header: '' },
]

// Create form
const billingOptions = ['PIX', 'BOLETO', 'CREDIT_CARD', 'UNDEFINED'] as const

function plusDays(n: number) {
  const d = new Date()
  d.setDate(d.getDate() + n)
  return d.toISOString().slice(0, 10)
}

const createSchema = z.object({
  value: z.number().positive('Must be > 0'),
  dueDate: z.string().min(1, 'Required'),
  billingType: z.enum(billingOptions),
  description: z.string().optional(),
})
type CreateSchema = z.output<typeof createSchema>

const createState = reactive<Partial<CreateSchema>>({
  value: undefined,
  dueDate: plusDays(3),
  billingType: 'PIX',
  description: undefined,
})
const creating = ref(false)

async function onCreate(event: FormSubmitEvent<CreateSchema>) {
  creating.value = true
  const { data, error } = await auth.asaas.payments({
    value: event.data.value,
    dueDate: event.data.dueDate,
    billingType: event.data.billingType,
    description: event.data.description,
  })
  creating.value = false

  if (error || !data) {
    toast.add({ title: 'Create failed', description: error?.message, color: 'error' })
    return
  }
  paymentIds.value = [data.id, ...paymentIds.value]
  await refreshAll()
  createState.value = undefined
  createState.description = undefined
  toast.add({ title: 'Payment created', description: data.id, color: 'success' })
}

async function signOut() {
  await auth.signOut()
  await navigateTo('/login', { replace: true })
}
</script>

<template>
  <div class="mx-auto max-w-4xl p-8">
    <header class="mb-8 flex items-center justify-between">
      <h1 class="text-2xl font-semibold">
        Payments
      </h1>
      <div class="flex items-center gap-3 text-sm">
        <span>{{ session?.user?.email }}</span>
        <UButton variant="outline" @click="signOut">
          Sign out
        </UButton>
      </div>
    </header>

    <section class="mb-10">
      <h2 class="mb-3 text-lg font-medium">
        Create charge
      </h2>
      <UForm :schema="createSchema" :state="createState" class="grid grid-cols-2 gap-4" @submit="onCreate">
        <UFormField label="Value (BRL)" name="value" required>
          <UInput v-model.number="createState.value" type="number" step="0.01" class="w-full" />
        </UFormField>

        <UFormField label="Due date" name="dueDate" required>
          <UInput v-model="createState.dueDate" type="date" class="w-full" />
        </UFormField>

        <UFormField label="Billing type" name="billingType" required>
          <USelect v-model="createState.billingType" :items="billingOptions" class="w-full" />
        </UFormField>

        <UFormField label="Description" name="description">
          <UInput v-model="createState.description" class="w-full" />
        </UFormField>

        <div class="col-span-2">
          <UButton type="submit" :loading="creating">
            Create
          </UButton>
        </div>
      </UForm>
    </section>

    <section>
      <div class="mb-3 flex items-center justify-between">
        <h2 class="text-lg font-medium">
          Your payments
        </h2>
        <UButton variant="ghost" icon="i-lucide-refresh-cw" :loading="tableLoading" @click="refreshAll">
          Refresh
        </UButton>
      </div>

      <UTable :data="rows" :columns="columns" :loading="tableLoading">
        <template #actions-cell="{ row }">
          <div class="flex gap-1">
            <UButton size="xs" variant="ghost" icon="i-lucide-refresh-cw" @click="refreshRow(row.original.id)" />
            <UButton size="xs" variant="ghost" color="error" icon="i-lucide-trash-2" @click="deleteRow(row.original.id)" />
          </div>
        </template>
      </UTable>

      <p v-if="!rows.length && !tableLoading" class="mt-4 text-sm text-neutral-500">
        No payments yet. Create one above.
      </p>
    </section>
  </div>
</template>
```
